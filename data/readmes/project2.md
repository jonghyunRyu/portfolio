# DaMoono - AI 상담 요약 통신 플랫폼

## 🔗 배포 URL
https://da-moono-frontend-deploy.vercel.app

## 📝 프로젝트 요약

실시간 상담과 AI 기반 상담 요약을 결합한 **통합 요금제 상담 플랫폼 프론트엔드 프로젝트**입니다.

사용자가 AI 챗봇 상담 → 실시간 상담사 연결 → 상담 요약 확인까지  
**하나의 흐름으로 경험할 수 있는 통신사 상담 서비스**를 구축했습니다.

- 역할: Frontend 개발
- 기술 스택: React, TypeScript, Vanilla-Extract, Three.js, Framer Motion, Chart.js

---

# 📖 프로젝트 설명

기존 통신사 상담 서비스에서는 AI 챗봇, 실시간 상담, 상담 이력 확인 기능이 **각각 분리되어 있어**  
상담 흐름이 끊기고 사용자 경험이 불편한 문제가 있었습니다.

이를 해결하기 위해 다음 목표로 통합 상담 플랫폼을 개발했습니다.

- AI 챗봇 → 실시간 상담 → 상담 요약까지 **하나의 서비스 흐름으로 통합**
- 상담 내용을 **UI 친화적인 JSON 구조로 요약하여 렌더링**
- 상담 이력과 이용 데이터를 **시각화하여 제공**
- 사용자 경험을 고려한 **로딩 최적화 및 인터랙티브 UI 구현**

# 🎯 문제 상황

## 1. 배포 환경에서 이미지 로드 실패 문제

로컬 개발 환경에서는 정상적으로 표시되던 이미지가  
**Vercel 배포 환경에서 로드되지 않는 문제**가 발생했습니다.

원인을 분석한 결과, 이미지 경로 참조 방식이  
빌드 과정에서 변환되면서 **경로가 깨지는 것**이 원인이었습니다.

---

## 2. 404 페이지 사용자 경험 문제

SPA 구조에서 잘못된 경로로 접근했을 때  
**사용자에게 아무런 안내 없이 빈 화면이 표시되는 문제**가 있었습니다.

사용자가 길을 잃었을 때 **자연스럽게 서비스로 돌아올 수 있는 경험**이 필요했습니다.

---

# 💡 해결 방법

## 1. 배포 환경 이미지 경로 문제 해결

빌드 환경에서 이미지 경로가 깨지는 문제를 분석하고 해결했습니다.

해결 과정

- 상대 경로 참조 방식에서 **import 기반 정적 자산 참조 방식**으로 변경
- Vite의 정적 자산 처리 방식을 활용하여 **빌드 시 해시된 경로로 자동 변환**되도록 수정
- public 폴더와 src 내부 자산의 **참조 방식 차이를 정리**하고 팀 내 공유

효과

- 로컬과 배포 환경 모두에서 **이미지가 정상적으로 로드**
- 팀 전체의 **정적 자산 관리 방식 통일**

---

## 2. 404 페이지 설계 및 구현

사용자가 잘못된 경로로 접근했을 때 **자연스럽게 서비스로 돌아올 수 있는 404 페이지**를 구현했습니다.

구현 내용

- 서비스 톤앤매너에 맞는 마스코트 일러스트 및 메시지 구성
- 사용자 역할(ADMIN/USER)에 따른 **동적 리다이렉트 경로** 설정
- `createBrowserRouter`의 `path: '*'` 와일드카드로 모든 미등록 경로 처리
